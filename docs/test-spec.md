# Reel Creator - テスト仕様書

## 1. 概要

### 1.1 プロジェクト概要
Reel Creator は TikTok / Instagram Reels 用の縦型動画から無音部分を自動削除し、AI 字幕を付与する Web アプリケーションです。

**技術スタック:**
- フロントエンド: Next.js 16 + Tailwind CSS v4
- バックエンド: FastAPI + FFmpeg + faster-whisper
- インフラストラクチャ: Docker Compose

### 1.2 テスト方針

| テスト層 | 手法 | 実施者 | 備考 |
|---------|------|------|------|
| ユニットテスト（バックエンド） | pytest | 自動実行 | Python による単体テスト |
| API テスト | pytest + FastAPI TestClient | 自動実行 | エンドポイント検証 |
| 統合テスト | 手動 | QA | Docker Compose で全体起動 |
| パフォーマンステスト | 手動 | QA | 処理時間計測 |

### 3.1 AT-001: POST /api/upload - ファイルアップロード

|---|---|---|---|---|---|
| AT-001-01 | 正常アップロード（MP4） | 有効な MP4 ファイル (30秒、10MB以下) | 200 | `{job_id, filename, duration, file_size}` | 成功ケース |

| # | テストケース名 | 入力 | HTTP ステータス | レスポンス | 備考 |
|---|---|---|---|---|---|

| AT-002-02 | 正常処理開始（字幕あり） | 有効な job_id + 字幕設定 | 200 | `{job_id, status: "processing"}` | 字幕処理開始 |

### 3.3 AT-003: GET /api/progress/{job_id} - 処理状況監視

**目的:** Server-Sent Events (SSE) による処理進捗のストリーミング配信の検証

**エンドポイント:** `GET /api/progress/{job_id}`

**接続方法:** SSE (Server-Sent Events)

| # | テストケース名 | 入力 | 期待結果 | 備考 |
|---|---|---|---|---|
| AT-003-01 | SSE 正常配信 | 処理中の job_id | イベントが順次配信される | 進捗 0% → 100% |
| AT-003-02 | 段階的進捗更新 | 処理中の job_id | `data: {progress: 0, message: ""}` → `{progress: 50, message: "..."}` → `{progress: 100, message: "完了"}` | 複数イベント送信 |
| AT-003-03 | 存在しないジョブ | 無効な job_id | 404 | ジョブ存在確認 |
| AT-003-04 | 完了後 SSE 接続 | 完了済み job_id | 最終進捗のみ返却または接続終了 | 完了状態確認 |

**クライアント実装例（JavaScript）:**
```javascript
const eventSource = new EventSource('/api/progress/abc123def456');
eventSource.onmessage = (event) => {
  console.log(JSON.parse(event.data));
  // {progress: 50, message: "..."}
};
```
