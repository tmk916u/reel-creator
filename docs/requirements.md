# Reel Creator 要件定義書

**バージョン:** 1.0.0
**作成日:** 2026年2月25日
**ステータス:** 確定

---

## 1. プロジェクト概要

### 1.1 プロジェクト名
Reel Creator

### 1.2 プロジェクト目的
TikTok・Instagramリール配信用の動画を簡単に作成するWebアプリケーションの開発。ユーザーは動画ファイルをアップロードし、無音部分の自動削除やAI字幕自動生成などの機能を活用して、短編動画コンテンツを効率的に制作できる。

### 1.3 対象ユーザー
- 動画コンテンツ制作者（個人〜小規模チーム）
- TikTokやInstagramなどのショートフォーム動画プラットフォームに定期的にコンテンツを配信する者

### 1.4 ビジネス価値
- 動画編集作業の効率化により、制作者のコンテンツ生産速度を向上
- 無音部分の自動削除により、視聴者体験を向上
- AI字幕自動生成により、アクセシビリティと視聴率を向上

## 2. 機能要件

### FR-001: 動画アップロード

#### 説明
ユーザーが動画ファイルをシステムにアップロードする機能。

#### 詳細要件
- **アップロード方式:** ドラッグ&ドロップまたはファイル選択ダイアログ
- **対応ファイル形式:** MP4、MOV、WebM
- **最大ファイルサイズ:** 500MB
- **最大動画長:** 3分（180秒）
- **必須アスペクト比:** 9:16（縦型）
  - 入力時点で9:16のアスペクト比を持つ動画のみを受け入れる
  - その他のアスペクト比の動画はエラーメッセージを表示
- **アップロード後:** プレビュー画面で動画を再生可能な状態で表示
- **バリデーション:** ファイルタイプ、サイズ、アスペクト比、動画長をサーバー側で検証

#### 受け入れ基準
- ドラッグ&ドロップでファイルを選択できる
- ファイル選択ダイアログでファイルを選択できる
- 対応形式のファイルが正常にアップロードされる
- 非対応形式、超過サイズ、不正なアスペクト比のファイルは適切なエラーメッセージを表示
- アップロード後、プレビューで動画が再生可能

---

### FR-002: 無音部分の自動削除

#### 説明
アップロードされた動画から無音区間を自動検出し、有音部分のみを抽出して結合した新しい動画を生成する機能。

#### 詳細要件
- **検出方式:** FFmpeg silencedetectフィルタを使用
- **ユーザー設定可能パラメータ:**
  - **無音の閾値（dB）:** -50dB〜-10dB（デフォルト: -30dB）
  - **最小無音長（秒）:** 0.1秒〜3.0秒（デフォルト: 0.5秒）
- **処理内容:** 検出された無音区間を除去し、有音区間のみを時系列順に結合
- **出力:** MP4形式の動画ファイル

#### 受け入れ基準
- 無音区間を正確に検出できる
- ユーザーが閾値と最小無音長をカスタマイズできる
- 処理後の動画で有音区間が正確に保持される
- 処理後の動画が再生可能な形式で生成される

---

### FR-003: AI字幕自動生成（オプション）

#### 説明
アップロードされた動画の日本語音声を自動的に文字起こしして、字幕を生成し、動画に焼き込む機能。処理はオプションであり、ユーザーが有効化を選択した場合のみ実行される。

#### 詳細要件
- **音声認識エンジン:** faster-whisper（base modelを使用）
- **言語:** 日本語
- **実行環境:** CPU（int8量子化）
- **出力形式:** SRT形式で字幕ファイルを一旦生成し、その後FFmpegで動画に焼き込み
- **ユーザー設定可能パラメータ:**
  - **フォントサイズ:** 小・中・大の3段階
  - **字幕位置:** 下部・中央の2種類
  - **文字色:** 白・黄色の2種類
- **処理フロー:** オプション有効化 → 字幕生成 → 動画への焼き込み
- **出力:** MP4形式の動画ファイル（字幕焼き込み済み）

#### 受け入れ基準
- 日本語音声を正確に文字起こしできる
- SRT形式の字幕ファイルが正確に生成される
- 字幕がユーザーの設定に従い、正確に動画に焼き込まれる
- 字幕テキストが選択した言語で正確に表示される

---

### FR-004: リアルタイム進捗表示

#### 説明
動画処理中の進捗状況をリアルタイムでユーザーに提示する機能。

#### 詳細要件
- **技術:** Server-Sent Events（SSE）でサーバーからクライアントへ進捗情報をストリーム配信
- **表示内容:**
  - 現在の処理ステージを明示（例: "動画をアップロード中"、"無音部分を検出中"、"字幕を生成中"）
  - 進捗率（0-100%）をプログレスバーで表示
  - 推定所要時間（もしあれば）
- **ユーザーエクスペリエンス:** 処理中は画面遷移を禁止し、キャンセル機能を提供可能

#### 受け入れ基準
- SSE接続が確立され、リアルタイムで更新される
- 各処理ステージのメッセージが正確に表示される
- プログレスバーが0-100%の範囲で正確に更新される
- 網接続の切断後も復帰可能である（リトライロジック実装）

---

### FR-005: 処理結果表示とダウンロード

#### 説明
動画処理の完了後、結果を表示し、ユーザーが処理済み動画をダウンロードできる機能。

#### 詳細要件
- **結果プレビュー:** 処理済み動画をWebプレイヤーで再生可能な状態で表示
- **処理結果サマリー:** 以下の情報を表示
  - 元の動画の長さ（秒）
  - 処理後の動画の長さ（秒）
  - 削除された無音の合計（秒）
  - 削除率（%）
- **ダウンロード機能:**
  - MP4形式で動画をダウンロード可能
  - ファイル名に処理日時やプロジェクト名を含める
- **アクション:**
  - 「ダウンロード」ボタンで処理済み動画をダウンロード
  - 「もう1本作る」ボタンでアップロード画面に戻る

#### 受け入れ基準
- 処理済み動画がプレビューで正常に再生される
- サマリー情報が正確に表示される
- ダウンロード機能が正常に動作する
- 「もう1本作る」ボタンで最初の画面に戻される

## 3. 非機能要件

### NFR-001: パフォーマンス

#### 目標値
- **3分の動画処理時間:**
  - 字幕なし（無音削除のみ）: 2分以内
  - 字幕あり（無音削除+AI字幕生成）: 5分以内
  - ※これらは目安値であり、サーバーのリソースに依存

- **ファイルアップロード時間:**
  - 100MBのファイル: 10秒以内（ローカル環境）
  - ネットワーク速度に依存するため、本番環境では異なる場合がある

#### 実装上の考慮
- 非同期処理（FastAPI + Celery相当の仕組み）を採用
- 大規模ファイルはチャンク分割でアップロード
- キャッシング機構の検討

---

### NFR-002: セキュリティ

#### CORS（Cross-Origin Resource Sharing）
- **許可するオリジン:** フロントエンドのオリジン（デフォルト: `http://localhost:3000`）
- **許可するメソッド:** GET、POST、OPTIONS
- **その他のオリジンからのアクセスはブロック**

#### 一時ファイル管理
- **保持期間:** ジョブ完了から1時間後
- **削除方法:** スケジュール化された自動削除ジョブで削除
- **保存位置:** `/tmp`またはDocker内の一時ディレクトリ

#### バリデーション
- **ファイルタイプの検証:** MIMEタイプとファイル拡張子を検証
- **ファイルサイズの検証:** 最大500MBを超えるファイルはアップロード時点で拒否
- **アスペクト比の検証:** FFprobeで取得した解像度から計算し、9:16比率であることを確認

#### その他セキュリティ対策
- HTTPS通信の推奨（本番環境）
- リクエストレート制限の実装
- ユーザー入力のサニタイズ

---

### NFR-003: 拡張性

#### 認証機能の後追加対応
- **現状:** 認証なしで全ユーザーが利用可能（パブリックAPI）
- **将来対応:** 以下の要素を考慮した設計
  - FastAPI Depends()を使用したミドルウェア層の構築
  - トークン（JWT等）ベースの認証機構
  - ユーザーセッション管理

#### クラウドデプロイ対応
- **現状:** Docker Composeで構成
- **将来対応:**
  - Kubernetes（k8s）への移行を想定
  - コンテナイメージのレジストリ（Docker Hub、ECR等）への対応
  - 環境変数による設定の外部化
  - ボリューム管理の柔軟化（ローカルマウント→クラウドストレージ）

---

### NFR-004: 運用性

#### デプロイ
- **起動方法:** `docker-compose up` で一発起動
- **設定ファイル:** `docker-compose.yml`で全サービスの構成を管理

#### ロギング
- **ログ出力レベル:** DEBUG、INFO、WARNING、ERROR、CRITICALに対応
- **環境変数:** `PYTHONUNBUFFERED=1`を設定し、ログをリアルタイムで出力
- **ログフォーマット:** タイムスタンプ、ログレベル、メッセージを含む標準形式

#### ヘルスチェック
- **エンドポイント:** `GET /api/health`
- **レスポンス:** JSON形式で以下の情報を返す
  ```json
  {
    "status": "ok",
    "timestamp": "2026-02-25T12:34:56Z"
  }
  ```
- **用途:** ロードバランサー、Kubernetes、監視ツールから定期的にヘルスチェック

## 5. ユーザーインターフェース

### 画面一覧

#### Step 1: アップロード画面
- **目的:** 動画ファイルをアップロード
- **内容:**
  - ドラッグ&ドロップエリア（または「ファイルを選択」ボタン）
  - アップロード可能なファイル形式、サイズ、動画長などの制限事項を表示
  - ファイル選択後、次のステップへ遷移するボタン

  - **AI字幕自動生成（オプション）:**
    - 有効化チェックボックス
    - フォントサイズ選択（小・中・大）
    - 字幕位置選択（下部・中央）
    - 文字色選択（白・黄色）

- **目的:** 処理進捗のリアルタイム表示
- **内容:**
  - 現在の処理ステージを表示
  - プログレスバー（0-100%）
  - 推定所要時間（オプション）
  - キャンセルボタン（オプション）

#### Step 4: 完了・ダウンロード画面
- **目的:** 処理結果の表示とダウンロード
- **内容:**
  - 処理済み動画のプレビュー再生
  - **処理結果サマリー:**
    - 元の動画の長さ
    - 処理後の動画の長さ
    - 削除された無音の合計
    - 削除率（%）
  - 「ダウンロード」ボタン
  - 「もう1本作る」ボタン

## 6. API仕様

### API エンドポイント一覧

| メソッド | パス | 説明 | リクエスト | レスポンス |
|---------|------|------|-----------|----------|
| GET | `/api/health` | ヘルスチェック | なし | JSON（status, timestamp） |
| POST | `/api/upload` | 動画アップロード | multipart/form-data | JSON（job_id, file_name） |
| POST | `/api/process/{job_id}` | 処理開始 | JSON（設定パラメータ） | JSON（status, message） |
| GET | `/api/progress/{job_id}` | 進捗SSEストリーム | なし | text/event-stream |
| GET | `/api/result/{job_id}` | 処理結果取得 | なし | JSON（サマリー情報） |
| GET | `/api/download/{job_id}` | 動画ダウンロード | なし | MP4ファイル |

### 各エンドポイントの詳細

#### GET /api/health
**説明:** バックエンドのヘルスチェック

**レスポンス例:**
```json
{
  "status": "ok",
  "timestamp": "2026-02-25T12:34:56Z"
}
```

---

#### POST /api/upload
**説明:** 動画ファイルをアップロード、一時保存し、ジョブIDを返す

**リクエスト:**
- Content-Type: `multipart/form-data`
- パラメータ:
  - `file`: 動画ファイル（MP4、MOV、WebM）

**レスポンス例:**
```json
{
  "job_id": "abc123def456",
  "file_name": "sample_video.mp4",
  "file_size": 52428800,
  "duration": 150,
  "width": 1080,
  "height": 1920,
  "aspect_ratio": "9:16"
}
```

**エラーレスポンス例:**
```json
{
  "error": "Invalid file type",
  "detail": "Only MP4, MOV, and WebM are supported"
}
```

---

#### POST /api/process/{job_id}
**説明:** 動画処理を開始

**パス パラメータ:**
- `job_id`: アップロード時に返却されたジョブID

**リクエスト例:**
```json
{
  "remove_silence": true,
  "silence_threshold": -30,
  "min_silence_duration": 0.5,
  "add_subtitles": true,
  "subtitle_font_size": "medium",
  "subtitle_position": "bottom",
  "subtitle_text_color": "white"
}
```

**レスポンス例:**
```json
{
  "status": "processing",
  "job_id": "abc123def456",
  "message": "Processing started"
}
```

---

#### GET /api/progress/{job_id}
**説明:** 処理進捗をServer-Sent Events（SSE）でストリーム配信

**パス パラメータ:**
- `job_id`: 処理中のジョブID

**レスポンス例（複数の更新が送信される）:**
```
data: {"stage": "uploading", "progress": 10}

data: {"stage": "detecting_silence", "progress": 25}

data: {"stage": "removing_silence", "progress": 50}

data: {"stage": "generating_subtitles", "progress": 75}

data: {"stage": "finalizing", "progress": 100}

data: {"stage": "completed", "progress": 100, "result_id": "result123"}
```

---

#### GET /api/result/{job_id}
**説明:** 処理完了後の結果サマリーを取得

**パス パラメータ:**
- `job_id`: 処理完了したジョブID

**レスポンス例:**
```json
{
  "job_id": "abc123def456",
  "original_duration": 150,
  "processed_duration": 95,
  "silence_removed": 55,
  "silence_removal_rate": 36.67,
  "subtitles_added": true,
  "file_name": "sample_video_processed.mp4"
}
```

---

#### GET /api/download/{job_id}
**説明:** 処理済み動画をMP4形式でダウンロード

**パス パラメータ:**
- `job_id`: 処理完了したジョブID

**レスポンス:**
- Content-Type: `video/mp4`
- Content-Disposition: `attachment; filename="sample_video_processed.mp4"`
- ボディ: MP4ファイルのバイナリデータ

## 8. 制約事項および前提条件

### システム前提条件
- バックエンドサーバーに十分なCPUリソースが必要（AI字幕生成のため）
- 十分なディスク容量が必要（一時ファイル、処理済み動画の保存用）

### 制約事項
- **同時処理数:** ハードウェアのリソースに応じて決定
- **処理時間の目安:** 実際の処理時間はサーバーのスペックやネットワーク状況に依存
- **字幕精度:** faster-whisperの性能に依存し、100%の正確性は保証されない

### 将来の改善予定
- 認証機能の追加
- ユーザー登録・ログイン機能
- 処理履歴の保存・管理
- カスタムフォントの対応
- より高度な動画編集機能（エフェクト、トランジションなど）
- マルチスレッド・分散処理への対応

## 付録A: 用語集

| 用語 | 定義 |
|------|------|
| **アスペクト比** | 映像の幅と高さの比率（例: 9:16は縦型） |
| **dB（デシベル）** | 音量を表す単位。値が小さいほど静か |
| **FFmpeg** | オープンソースの動画・音声処理ツール |
| **faster-whisper** | OpenAIのWhisperモデルを高速化した音声認識エンジン |
| **SSE（Server-Sent Events）** | サーバーからクライアントへ単方向で情報をリアルタイム配信する技術 |
| **SRT形式** | SubRip Text形式。字幕ファイルの標準フォーマット |
| **CORS** | Cross-Origin Resource Sharing。異なるオリジン間のリソースアクセスを制御 |
| **job_id** | 処理ジョブに割り当てられた一意の識別子 |
| **焼き込み** | 字幕などの要素を動画に直接合成して、分離できない状態にすること |
