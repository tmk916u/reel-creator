## Why

`fix-third-stage-asr-leading-miss` 適用後のベースライン (job f4703bef) で:

| 項目 | 現状 | 目標 |
|------|------|------|
| #7 助詞直後切れ Dialogue 比率 | 38.7% | 10% 以下 (SHALL) |
| #8 字幕 8-14 文字 比率 | 41.9% | 70% 以上 (SHALL) |

具体的に出力字幕で観測されている問題:

```
Dialogue: 結論から言うと一番大事なのは     ← 「は」で flush(助詞切れ)、 続きが「メンタルです」
Dialogue: メンタルです。
Dialogue: 目標に向けて食事をどういうふう   ← 「ふう」で切れて、 続きが「に」
Dialogue: にしていこうというふうに考えた   ← 「考えた」で切れて、 続きが「うえで」
Dialogue: うえで実際にそのとおりできる方   ← 助詞「で」「に」が中に
```

原因分析:
- `_trailing_particles = "はがをにでとのもへやかな"` 11 文字しか抑制対象になく、 「て / で / ば / し / か」 等の接続助詞や活用語尾末が抜けている
- 「、」 で常に flush するため短い Dialogue (1-7 文字) が頻発
- `_merge_short_segments` の発動条件 (両方 < 0.6秒 or どちらかが < 5 文字) が緩く、 5-7 文字台の Dialogue が独立したまま残る

業務量産品質ライン 14 項目のうち最後の不合格 2 項目で、 これを直せば 業務量産投入可能ラインに到達(目視 9/14 → 11/14 見込み)。

## What Changes

- **助詞抑制の拡張**: `_trailing_particles` に接続助詞・活用語尾を追加
  - 旧: `はがをにでとのもへやかな` (11 文字)
  - 新: `はがをにでとのもへやかなてばしず` (15 文字、 「て / ば / し / ず」 を追加)
  - 副助詞 2-3 文字パターン (「から」 「ので」 「のに」 「けど」) は別途末尾チェックで対応
- **「、」 flush の緩和**: 「、」 では flush しない (文の区切りは句点だけにする)
  - 旧: `if text and text[-1] in "、。！？!?.":`
  - 新: `if text and text[-1] in "。！？!?.":` (「、」 を除外)
- **短い Dialogue の積極的統合**: `_merge_short_segments` を強化
  - 旧: 両方が短時間 + 合計が短い、 もしくは どちらかが < 5 文字
  - 新: どちらかが < 8 文字 → 統合候補。 合計 max_chars × 1.4 (= 14) まで結合許容
- 既存 API 不変、 BREAKING なし
