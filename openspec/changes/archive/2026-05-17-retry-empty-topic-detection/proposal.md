## Why

トピックラベル生成が確率的に 0 件 (空配列) を返すケースが頻発 (job 9437d999 で 4/4 すべて空、 過去複数の動画で #3 のみ空)。

`detect_topics` の LLM プロンプトには「動画が短すぎる or 単一トピックの場合は分割しない」 とあり、 LLM が「分割不要」 と判断すると `{"topics": []}` を返す。 結果、 動画にトピックテロップが何も出ない → 物足りない印象。

業務量産で「HOOK + トピック + 字幕 + CTA」 の 4 要素が揃っていることが投稿クオリティの目安。 トピック 0 件はこの揃いを崩す。

## What Changes

- `detect_topics` にリトライ機構を追加:
  - 1 回目: 現行プロンプト (`_TOPICS_SYSTEM_PROMPT`)
  - 結果が 0 件なら新規 `_TOPICS_FORCE_PROMPT` で 1 回リトライ
  - リトライプロンプトで「**最低 2 件、 最大 4 件で必ず分割**」 を強調
- BREAKING: なし

## Capabilities

### Modified Capabilities
- `quality-line`: 項目#12 トピックラベル整合 の安定性向上

## Impact

- **Backend**: llm.py の detect_topics に retry ロジック (約 15 行)、 新規 prompt 1 つ
- **テスト**: 既存 132 件 + 新規 2 件 (リトライ動作)
- **業務量産**: トピックラベルが確実に 2-4 件揃う
- **副作用**:
  - LLM 呼出が最大 2 回 (現在 1 回) → API コスト微増
  - リトライ発動率は ~10% (確率的失敗ケース) なので影響小
